# Scheduler library

The extraction of Koan's runtime substrate — scheduler, region memory, witnessed
carriers — into one Koan-agnostic library whose public surface is memory-safe by
construction. North star: [design/scheduler-library.md](../../design/scheduler-library.md).
This project's items land the consumer API and the boundary moves incrementally;
each item is sized for one PR. Objectives not yet scoped as items are listed
under "Not yet planned" below.

## Not yet planned

Objectives from [design/scheduler-library.md](../../design/scheduler-library.md)
not yet scoped as roadmap items — listed here so the project's full arc is
visible; each becomes one or more items when its prerequisites are close.

- **The opaque reach set** (guarantee 2). Reach-set minting becomes
  library-owned: an opaque type constructible only from region handles and
  carriers, with infallible union/transfer (deleting the hand-asserted
  totality `expect`s), while Koan's reach policy queries it through
  predicates. `FrameSet`'s current role migrates behind this type.
- **The step construction context** (guarantees 3 and 5 — the witnessed
  hoist proper). Finishes receive `ctx.alloc` / `ctx.alloc_with` and build
  outputs only through them: dep payloads viewable only at the closure
  brand, reach folded by construction, purity structural. Includes migrating
  builtin finishes off the `relocate_values` adapter (the `AwaitContinue`
  signature stays as-is until this tranche — the dep-finish delivery loops
  are already unified onto one `TerminalDepFinish` currency; see
  [design/scheduler-library.md](../../design/scheduler-library.md) § The
  consumer API) and routing the born-pure `seal_value(…, None)` terminal
  sites through the context.
- **Regions wholesale; `Scope` as a naming layer.** Scope storage allocates
  through library region handles held by `CallFrame`; the at-will (non-step)
  allocation surface re-homes onto the library with no koan-side arena
  ownership left.
- **Koan protocol combinators.** The recurring builtin protocols above
  `Action` — resolve-a-type-or-await-its-producer with re-resolve-on-wake
  (hand-rolled today in `newtype_def.rs` and `fn_def/return_type.rs`) —
  become named combinators.
- **Ambient bracket hygiene.** Closure-scoped or Drop-backed brackets for
  the per-step ambient state (`SlotStepGuard`, `swap_active_frame`,
  `active_in_contract_chain`) — explicitly deferred by
  [The step owns region liveness](step-held-region.md).
- **Actual extraction.** Once the sub-crate boundary holds and the API
  stabilizes: naming, docs, and publishing the substrate for use outside
  this repo.
