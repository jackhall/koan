carriers — into Koan-agnostic library crates whose public surface is memory-safe
by construction: the `workcell` computation-cell substrate (working name —
[design/workcell.md](../../design/workcell.md)) beneath the `workgraph` DAG
scheduler. North star: [design/scheduler-library.md](../../design/scheduler-library.md).

### Under-pinning (dangle-capable)

- **Fold-closure capture provenance.** A closure under a reach-folding
  combinator (`transfer_into` / `merge_pinned` / `map_pinned` in
  [workgraph/src/witnessed.rs](../../workgraph/src/witnessed.rs),
  `alloc_carried_with` in
  [arena.rs](../../src/machine/core/arena.rs)) is covered only for values
  built from the closure's own operands — but a `move` closure can capture
  any outside reference and embed it in the built value. Everything unifies
  at the brand lifetime and compiles clean; the composed witness never names
  the smuggled borrow. The fold-capability type confines *where* folded
  placements happen, not *what* the closure captures. A fix needs either a
  capture-free closure discipline the compiler can see, or a
  restructuring so fold closures receive everything they build from.
- **Evidence-to-pin coupling.** The move-in audits verify a value's borrows
  against the evidence the caller *passed*; nothing verifies that this
  evidence is what ultimately pins the stored reference. The alloc and the
  pin (the later `resident_type_carrier` wrap, `bind_value` reach, or
  carrier fold) are separate acts paired by adjacency. Candidate shape:
  fused veneers where the mint/pin and the store are one call, so the
  pairing is by construction (the pattern of `Delivered::adopt_into` in
  [workgraph/src/witnessed/delivered.rs](../../workgraph/src/witnessed/delivered.rs)).
- **The bind leg is untyped.** `Scope::bind_value(name, object, index, reach)`
  and `Scope::resident_type_carrier(kt, reach, borrows_into_home)`
  ([scope.rs](../../src/machine/core/scope.rs)) accept a caller-supplied
  `StoredReach` / home-bit with no tie to the value's actual borrows; the
  `omit` predicates on reach mints are equally free-form. A wrong reach,
  bit, or omit under-pins, and a later read rebuilds a carrier from the
  stored lie. This is the binding-side twin of the move-in item: the same
  audit-or-evidence treatment applied where values *enter scopes* rather
  than where they enter regions.
- **The within-step transient invariant is temporal, not typed.** Empty-
  witness carriers (`RegionBrand::alloc_object_witnessed`,
  `Witnessed::resident` returns such as `Outcome::done_pure`) pin nothing;
  soundness rests on the active frame pinning the region for the
  construction step and finalize folding the producer before the carrier
  lands on a node ([arena.rs](../../src/machine/core/arena.rs) doc on
  `alloc_object_witnessed`). Safe code that stashes such a carrier past its
  step, or returns a foreign-borrowing value through a region-pure marker
  path, under-pins with no audit on the path.
- **Residence side-table blind spots.** The `owns_addr` table
  ([workgraph/src/witnessed/region.rs](../../workgraph/src/witnessed/region.rs))
  is bare-`usize` membership: (a) `KObject::Wrapped { type_id: &KType }` is
  unauditable because `KType` is deliberately un-recorded, so an object
  whose only foreign borrow is its type identity passes every runtime
  audit; (b) a freed region's address reused by another arena can
  false-pass a stale pointer (no generation/provenance); (c) recording is a
  per-family opt-in in the `Stored::record_local` impls
  ([arena.rs](../../src/machine/core/arena.rs)) — a future region-borrowing
  family that forgets to opt in silently becomes audit-permissive.
- **Library escape hatches.** `RegionHandle::alloc_resident_audited` is a
  safe `pub` workgraph method whose soundness rests entirely on the
  caller-supplied audit closure — `|_, _| true` reproduces the unvetted
  lifetime-lengthening move-in. Koan code can also reach a raw
  `RegionHandle` (`RegionBrand::handle()`) and bypass every koan veneer.
  Decide the published shape before the API freezes: embedder-trust
  documentation, `unsafe fn`, a per-family audit on the `Stored` trait, or
  unpublished. Likely folds into
  [Publishing the workgraph crate](workgraph-extraction.md).
- **The frame `outer`-chain protocol.** `build_frame_child_witnessed`
  ([arena.rs](../../src/machine/core/arena.rs)) keeps a per-call child's
  cross-region parent alive via `FrameStorage`'s `outer` `Rc` chain — an
  invariant upheld by one construction door and its caller wiring
  `outer_frame` correctly. A caller passing `None` for a per-call parent
  under-pins silently; no audit checks chain correctness.

### Over-pinning (leak / retention)

- **Cross-region pin cycles.** Reach sets hold `Rc<FrameStorage>` members.
  Home-omission breaks the self-cycle by construction, but a mutual pin —
  region A's set retaining frame B while region B's set retains frame A —
  is expressible in safe code, defeats the refcount-driven region free, and
  nothing detects it short of the Miri leak slate. Candidate: a debug-mode
  cycle detector over the region-set graph, or a structural rule making
  mutual pins unrepresentable.
- **Reach over-approximation.** Folding a dep a value did not actually
  borrow from keeps that dep's region alive as long as the carrier lives.
  The scalar gates in `alloc_type_with` / `alloc_object_with`
  ([arena.rs](../../src/machine/core/arena.rs)) counter the known cases;
  the general problem — every audit catches under-pinning only, and
  over-pinning always passes silently — has no observability. The wrong
  direction of the `borrows_into_home` bit (`true` when false) is the same
  hole: a retiring tail-call frame rides the fresh frame's bindings.
  Candidate: a debug-mode reach-tightness report (regions pinned vs regions
  actually referenced).
- **Side-table growth.** The residence `membership` vec records every
  recorded-family allocation address for the region's life and answers
  `owns_addr` by linear scan — cost that grows with exactly the audits the
  move-in item adds. Candidate: a sorted/hashed structure, or per-family
  tables sized to the sparse families that need them.
