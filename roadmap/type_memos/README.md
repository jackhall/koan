# Type memos

Content-addressed type identity, memoized dispatch, and modules as values — one
overhaul of `KType`'s identity and satisfaction story. Type identity becomes a wide
content-hash digest replacing raw-pointer nominal identity; subtype-match outcomes are
cached on that digest so overload resolution stops re-walking the same
`(value-type, slot-type)` pair; modules move to the value channel, typed by a
creation-time principal signature under a canonical signature-subtyping relation. The
end state is pinned in
[design/typing/module-values-and-type-identity.md](../../design/typing/module-values-and-type-identity.md).

- **Cross-thread type-content transfer.** The type registry is per-run-frame, so when
  future concurrency moves a value between threads, its types' content must land in
  the receiving frame's registry
  ([design/typing/type-registry.md](../../design/typing/type-registry.md),
  "Concurrency"). Two candidate mechanisms are recorded there — copy the value's type
  nodes plus everything reachable through their composition edges, skipping digests
  the receiver already holds; or, with persistent (immutable) node storage, merge the
  maps outright — and whether verdict edges transfer as warm cache is part of the same
  question. Handles need no translation under either: a digest is the same value in
  both registries. Exercisable only once concurrency ships.
