# Type memos

Content-addressed type identity, memoized dispatch, and modules as values — one
overhaul of `KType`'s identity and satisfaction story. Type identity becomes a wide
content-hash digest replacing raw-pointer nominal identity; subtype-match outcomes are
cached on that digest so overload resolution stops re-walking the same
`(value-type, slot-type)` pair; modules move to the value channel, typed by a
creation-time principal signature under a canonical signature-subtyping relation. The
end state is pinned in
[design/typing/module-values-and-type-identity.md](../../design/typing/module-values-and-type-identity.md).

## Unplanned work

Known gaps adjacent to this project's items, recorded so they aren't lost. **To start one,
first promote it into its own `roadmap/type_memos/<item>.md` file** with `**Problem.**` /
`**Acceptance criteria.**` / `## Dependencies` sections (see the documentation skill's
"When adding a new roadmap item"), then replace its prose entry here with a link. These
paragraphs name the gap, not the fix.

- **Module element-type join.** With modules riding the value channel as
  [`KObject::Module`](../../design/typing/modules.md#first-class-modules), a container of module
  values memoizes its element type as the join of the members' self-sigs — and `KType::join` of
  two distinct signatures falls to `Any`, so a heterogeneous module list never satisfies a
  `:(LIST OF Ordered)` slot even when every member satisfies `Ordered`. A principled join
  would meet at a common signature supertype over the canonical subtyping relation.
- **Interned type content behind `Copy` handles.** With digests shipped, structural
  content (and `RecursiveSet`) could move behind per-frame registry entries, making
  `KType` a `Copy` `{digest, registry}` handle: trivial clones, per-frame dedup of
  repeatedly built types, and a transitively-closed registry whose lift merge early-outs
  at any digest the parent already holds. Justified by measured clone cost, not assumed;
  reconciling `RecursiveSet`'s shared content is this item's nominal instance.
- **Value equality by rendering.** `Parseable::equal` compares `summarize()` output, so two
  distinct values with identical rendering compare equal under `==` — for modules, two
  same-path modules with different members. The pattern is shared by every `KObject` shape;
  a structural (or identity) equality ruling would replace the rendered-string compare.
